# 개발표준 샘플 (팀용 Markdown)

> 이 문서는 **사람이 읽고 합의하는 개발표준** 예시입니다.  
> Cursor 전용 규칙은 `standards/*.mdc` 및 `.cursor/rules/`를 사용하세요.

## 1. 적용 범위

- 저장소: `rag-orchestrator-agent`
- 언어: Python(백엔드), TypeScript(프론트), MCP 서버 모듈

## 2. 브랜치·커밋

- 기본 브랜치: `main` (또는 조직 표준)
- 기능: `feat/이슈-요약`, 수정: `fix/이슈-요약`
- 커밋 메시지: 한 줄 요약 + 필요 시 본문(무엇을/왜)

## 3. 코드 스타일

- Python: 포매터·린터는 팀이 선택한 도구에 따름 (예: Ruff).
- TypeScript: `npm run build`가 통과해야 함.
- **주석·커밋 설명**: 팀 언어 정책에 따름 (본 프로젝트는 한국어 우선).

## 4. 아키텍처 원칙

- RAG 저장소 접근은 **HTTP 핸들러에 직접 Chroma를 두지 않고**, MCP 도구를 경유한다.
- 에이전트 간 상태는 공유 `AgentState` 한 곳에서만 전달한다.
- 새 외부 연동은 인터페이스(클라이언트 모듈)로 감싼다.

## 5. 보안·비밀

- `.env`는 Git에 올리지 않는다. 예시만 `.env.example`로 둔다.
- 키·토큰은 로그·트레이스에 남기지 않는다.

## 6. 테스트·품질 게이트 (예시)

- PR 전: 관련 모듈 단위 테스트 또는 최소 수동 시나리오(README 절차) 통과.
- API 변경 시 `README.md`의 API 표를 갱신한다.

## 7. 문서 위치

| 개발표준(본 문서 형태) | `docs/standards/` |
| AI/에디터 규칙 | `standards/*.mdc`, `.cursor/rules/` |
