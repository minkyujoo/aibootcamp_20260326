# PRD — AI 영업관리 포탈 (영업활동 Assistant)

## 1. 문서 정보

| 항목 | 내용 |
|------|------|
| 제목 | AI 영업관리 포탈 — 잠재고객·사업기회·영업활동 통합 및 AI Assistant |
| 버전 | 1.0 |
| 상태 | 초안 |

## 2. 배경·목적

영업 사원이 잠재고객을 발굴하고 영업활동을 수행하며, 사업기회가 식별되면 해당 **고객사·담당자**를 대상으로 수주 가능성을 높이기 위한 활동을 이어간다. 본 제품은 이 과정에서 **활동 기록(메일·회의록·의견)을 쉽게 등록**하고, **고객사·사업기회와 자동 매핑**하며, **문의 응답·영업전략·Action item**을 AI Agent가 추천하는 **AI 영업관리 포탈**이다.

## 3. 목표·비목표

### 목표

- 영업활동을 단일 저장소에 모으고, **고객사·사업기회**와 신뢰 가능하게 연결한다.
- 담당자별·고객사별·기회별로 활동 이력을 조회한다.
- 등록된 활동과 단계 가이드를 바탕으로 **Action item·영업전략**을 추천한다.
- **수주확률**을 규칙·모델 기반으로 산출·갱신한다.

### 비목표 (초기 버전에서 제외하거나 후순위)

- 실제 메일 서버·ERP와의 양방향 실시간 동기화 (필요 시 추후 정의)
- data.go.kr 전 수집 파이프라인의 법적·운영 승인 대행 (연동 전제는 별도 검토)

## 4. 사용자·핵심 시나리오

| 페르소나 | 시나리오 |
|----------|----------|
| 영업 담당자 | 메일/회의록/의견을 등록하면 시스템이 고객사·사업기회를 제안·매핑하고, 목록에서 확인한다. |
| 영업 담당자 | 특정 고객사 또는 사업기회를 먼저 선택한 뒤 활동을 등록한다. |
| 영업 담당자 | 고객사·기회를 선택해 과거 활동을 보고, AI에게 관련 질문·전략·할 일을 요청한다. |
| 관리자(또는 팀장) | 영업 담당 10명을 추가·삭제하고, 담당자별 고객사·사업기회 배정을 조정한다. |

## 5. 데이터·외부 연동

### 5.1 고객사 마스터 (공공데이터)

- **출처**: 공공데이터포털 ([data.go.kr](https://www.data.go.kr)) API/데이터셋을 활용한다.
DART 정보를 활용해서 상세정보를 추가해줘.
- **범위**: **SK 멤버사 중심 20개사**만 초기 시드로 표시·동기화한다. (실제 API·항목·갱신 주기는 설계 단계에서 확정)
- 화면의 **고객사 목록**에는 위 기준으로 가져온 정보가 표시된다.

### 5.2 영업 담당자

- 영업 담당은 **10명**을 전제로 한다.
- **추가·삭제**가 가능하다.
- 담당자를 **선택**하면, 그 담당자에게 **매핑된 고객사·사업기회 목록**을 볼 수 있다.

### 5.3 고객사 ↔ 사업기회 ↔ 담당자 매핑

- 각 **고객사**·**사업기회**에는 **영업 담당**이 매핑되거나, 화면에서 **변경** 가능하다.
- 고객사와 연관된 **사업기회**·**영업활동**이 관계로 연결된다.

### 5.4 사업기회 정의

- 사업기회는 **회사(고객사)별**로 **3~5개** 수준으로 등록한다.
- 유형 예시: **ERP**, **그룹웨어**, **M365**, **CRM** 등 **SI 프로젝트** 성격의 기회.
- 기회마다 수주확률·단계·활동 이력을 관리한다.

## 6. 기능 요구사항

### 6.1 화면·정보 구조

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| UI-01 | **고객사 목록** 화면: data.go.kr 기반 20개사(SK 멤버사 중심) 정보 표시 | Must |
| UI-02 | **사업기회 목록** 화면: 고객사별 3~5개 SI 기회 표시 및 고객사와 연계 | Must |
| UI-03 | 고객사 선택 시 관련 **사업기회**·**영업활동** 매핑·요약 표시 | Must |
| UI-04 | 사업기회 선택 시 관련 **영업활동** 목록 표시 | Must |

### 6.2 영업활동 Assistant (등록·매핑)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| A-01 | **활동등록** UI: **메일 등록**, **회의록 등록**, **의견 등록** 메뉴(또는 탭) 제공 | Must |
| A-02 | 등록 시 본문·첨부(범위는 설계 시 정함)를 저장하고, **고객사·사업기회를 자동 판별·매핑** 제안 | Must |
| A-03 | 사용자가 **특정 고객사 또는 사업기회를 지정**한 상태에서 활동 등록 가능 | Must |
| A-04 | 매핑 결과는 사용자가 **수정·확정**할 수 있어야 함 (자동 오류 보정) | Should |

### 6.3 AI Agent (조회·추천)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| AI-01 | 선택한 **고객사 또는 사업기회**와 연결된 **등록 내용**을 컨텍스트로 **관련 문의에 답변** | Must |
| AI-02 | 동일 컨텍스트로 **영업전략** 제안 | Must |
| AI-03 | **Action item** 추천 | Must |
| AI-04 | **영업 활동 단계**별 가이드(사내 표준 또는 템플릿)를 참고해 단계에 맞는 **할 일·가이드** 제시 | Must |

### 6.4 수주확률

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| P-01 | 사업기회별 **수주확률** 산출·표시 | Must |
| P-02 | 확률 상향 요인 예: 영업활동 **빈도·최근성**, 활동 내 **고객 요구사항의 구체성**, **계약 구조·조건 협의** 언급 여부 등 | Must |
| P-03 | 산출 로직은 규칙 기반 점수 + (선택) LLM 요약 플래그 등으로 단계적 고도화 가능하도록 설계 | Should |

## 7. 메뉴 구성

1. **고객사** — 목록·상세·연결된 사업기회·활동·담당자  
2. **사업기회** — 목록·상세·수주확률·활동·담당자  
3. **영업담당** — 10명 관리(추가/삭제), 담당자 선택 시 담당 고객사·기회  
4. **활동등록** — 메일 / 회의록 / 의견 등록 및 매핑  

## 8. 비기능·제약

- 개인정보·메일 본문: 보관 기간·접근 권한·마스킹 정책은 보안/법무와 합의 후 반영.
- data.go.kr 이용 시 **이용약관·출처 표기** 준수.
- AI 응답은 **근거(어떤 활동·문서에서 왔는지)**를 가능한 한 추적 가능하게 표시하는 것을 권장.

## 9. 성공 지표 (예시)

- 활동 등록 후 **매핑 수정 없이 확정** 비율.
- 담당자별 **기회당 활동 수** 및 **수주확률 변화** 추이.
- Agent **Action item 채택률**(수동 피드백이 있으면 측정).
